# NomadNotes (游迹) 项目设计文档 v3.0

> 一个极简旅行记录全栈应用，帮助用户为每次旅程管理**账单**与**游记**，并在旅程结束时自动生成消费分析与时间线回顾。
> 技术栈：React 18 + TypeScript + Vite + Ant Design + Zustand + React Router + Axios  
> 后端：Node.js 20+ (ESM) + Express + MySQL + Sequelize + JWT 双 token + bcrypt + Multer

---

## 1. 项目概述

**核心定位**：轻量级旅行账本 + 游记工具，适合作为 React + Node.js 全栈实战项目，可部署上线并写入简历。

**目标用户**：希望记录旅行中每一笔开销与文字记忆的个人用户。

**主要功能**：

- 用户注册/登录（JWT 双 token 认证）
- 创建、编辑、删除旅程（设定目的地、日期、封面图）
- 在旅程中添加分类开支（金额、分类、备注、小票图片）
- 撰写游记（支持多图上传），按时间线查看
- **游记图片自动压缩**：上传的图片经 Canvas 压缩后持久化存储
- 旅程结束后生成统计：消费分类饼图、总支出、游记时间线
- 图片上传（小票、游记配图）

## 3. 数据库设计 (MySQL + Sequelize)

### 3.1 设计原则

- **禁止使用外键约束**（FOREIGN KEY）。表与表之间的关联全部通过字段名逻辑关联（如 `user_id`、`trip_id`），由应用层代码维护数据一致性。
- Sequelize 模型间可保留 `hasMany` / `belongsTo` 等关联，但必须配置 `constraints: false` 以避免在数据库中生成实际外键。
- 所有主键使用 UUID (CHAR(36))，由 Sequelize 自动生成。

### 3.2 表结构定义

#### 表 `users`

| 列名          | 类型         | 约束                    | 描述                          |
| ------------- | ------------ | ----------------------- | ----------------------------- |
| id            | CHAR(36)     | PRIMARY KEY, NOT NULL   | UUID 主键                     |
| username      | VARCHAR(50)  | UNIQUE, NOT NULL        | 登录用户名                    |
| email         | VARCHAR(255) | UNIQUE, NOT NULL        | 邮箱                          |
| password_hash | VARCHAR(255) | NOT NULL                | bcrypt 加盐 10 轮加密后的密码 |
| avatar_url    | VARCHAR(500) | NULL                    | 头像地址                      |
| created_at    | DATETIME     | NOT NULL, DEFAULT NOW() | 注册时间                      |

#### 表 `trips`

| 列名        | 类型         | 约束                    | 描述                    |
| ----------- | ------------ | ----------------------- | ----------------------- |
| id          | CHAR(36)     | PRIMARY KEY, NOT NULL   | UUID 主键               |
| user_id     | CHAR(36)     | NOT NULL                | 逻辑外键，关联 users.id |
| title       | VARCHAR(100) | NOT NULL                | 旅程名称                |
| destination | VARCHAR(100) | NOT NULL                | 目的地                  |
| start_date  | DATE         | NOT NULL                | 开始日期                |
| end_date    | DATE         | NOT NULL                | 结束日期                |
| cover_image | VARCHAR(500) | NULL                    | 封面图片路径            |
| is_ended    | TINYINT(1)   | NOT NULL, DEFAULT 0     | 是否已结束 (0/1)        |
| created_at  | DATETIME     | NOT NULL, DEFAULT NOW() | 创建时间                |

#### 表 `expenses`

| 列名          | 类型          | 约束                    | 描述                                |
| ------------- | ------------- | ----------------------- | ----------------------------------- |
| id            | CHAR(36)      | PRIMARY KEY, NOT NULL   | UUID 主键                           |
| trip_id       | CHAR(36)      | NOT NULL                | 逻辑外键，关联 trips.id             |
| category      | VARCHAR(20)   | NOT NULL                | 分类：餐饮/交通/住宿/购物/门票/其他 |
| amount        | DECIMAL(10,2) | NOT NULL                | 金额，精确到分                      |
| note          | TEXT          | NULL                    | 备注                                |
| receipt_image | VARCHAR(500)  | NULL                    | 小票图片路径                        |
| expense_date  | DATE          | NOT NULL                | 消费发生日期                        |
| created_at    | DATETIME      | NOT NULL, DEFAULT NOW() | 记录时间                            |

#### 表 `notes`

| 列名          | 类型     | 约束                    | 描述                                   |
| ------------- | -------- | ----------------------- | -------------------------------------- |
| id            | CHAR(36) | PRIMARY KEY, NOT NULL   | UUID 主键                              |
| trip_id       | CHAR(36) | NOT NULL                | 逻辑外键，关联 trips.id                |
| content       | TEXT     | NOT NULL                | 正文（支持 Markdown 或纯文本）         |
| images        | JSON     | NULL                    | 原始图片路径数组                       |
| vector_images | JSON     | NULL                    | 矢量 SVG 路径数组 (与 images 一一对应) |
| note_date     | DATE     | NOT NULL                | 记录日期，用于时间线排序               |
| created_at    | DATETIME | NOT NULL, DEFAULT NOW() | 创建时间                               |

**补充说明**：

- `images` 和 `vector_images` 使用 JSON 类型，Sequelize 中对应 `DataTypes.JSON`。
- `vector_images` 数组长度与 `images` 相同，`vector_images[i]` 为 `images[i]` 转换后的 SVG 路径，若转换失败则为 `null`。
- 级联删除由 service 层手动实现（先删子记录再删父记录）。

### 3.3 索引建议

```sql
CREATE INDEX idx_trips_user_id ON trips(user_id);
CREATE INDEX idx_expenses_trip_id ON expenses(trip_id);
CREATE INDEX idx_notes_trip_id ON notes(trip_id);
CREATE INDEX idx_expenses_category ON expenses(trip_id, category);
CREATE INDEX idx_expenses_date ON expenses(trip_id, expense_date);
CREATE INDEX idx_notes_note_date ON notes(trip_id, note_date);
```

### 3.4 Sequelize 模型定义示例 (ESM)

#### `server/src/models/User.js`

```javascript
import { DataTypes } from "sequelize";
import { sequelize } from "../config/database.js";

const User = sequelize.define(
	"User",
	{
		id: {
			type: DataTypes.UUID,
			defaultValue: DataTypes.UUIDV4,
			primaryKey: true,
		},
		username: {
			type: DataTypes.STRING(50),
			allowNull: false,
			unique: true,
			validate: { len: [2, 50] },
		},
		email: {
			type: DataTypes.STRING(255),
			allowNull: false,
			unique: true,
			validate: { isEmail: true },
		},
		passwordHash: {
			type: DataTypes.STRING(255),
			allowNull: false,
		},
		avatarUrl: {
			type: DataTypes.STRING(500),
			allowNull: true,
		},
		createdAt: {
			type: DataTypes.DATE,
			defaultValue: DataTypes.NOW,
		},
	},
	{
		tableName: "users",
		timestamps: false,
	},
);

export default User;
```

#### 关联定义 (`server/src/models/index.js`)

```javascript
import User from "./User.js";
import Trip from "./Trip.js";
import Expense from "./Expense.js";
import Note from "./Note.js";

// 所有关联均设置 constraints: false，禁止生成外键
User.hasMany(Trip, { foreignKey: "userId", constraints: false });
Trip.belongsTo(User, { foreignKey: "userId", constraints: false });

Trip.hasMany(Expense, { foreignKey: "tripId", constraints: false });
Expense.belongsTo(Trip, { foreignKey: "tripId", constraints: false });

Trip.hasMany(Note, { foreignKey: "tripId", constraints: false });
Note.belongsTo(Trip, { foreignKey: "tripId", constraints: false });

export { User, Trip, Expense, Note };
```

## 4. API 接口设计

### 4.1 基础信息

- **Base URL**: `http://localhost:3434/api`
- **认证方式**: Bearer Token，除注册/登录/刷新 token 外所有接口需在 Header 中携带 `Authorization: Bearer <accessToken>`
- **统一响应格式**:
  ```json
  {
  	"code": 200,
  	"message": "success",
  	"data": {}
  }
  ```
  错误时 `code` 为 HTTP 状态码，`message` 为错误描述。
- **分页格式**: 分页数据统一放在 `data` 对象内：
  ```json
  {
  	"code": 200,
  	"message": "success",
  	"data": {
  		"list": [],
  		"total": 100,
  		"page": 1,
  		"pageSize": 10
  	}
  }
  ```

### 4.2 鉴权相关（双 token）

#### 注册

- **POST** `/api/auth/register`
- **请求体**:
  ```json
  {
  	"username": "john_doe",
  	"email": "john@example.com",
  	"password": "securePassword123"
  }
  ```
- **成功响应** (201):
  ```json
  {
  	"code": 201,
  	"message": "注册成功",
  	"data": {
  		"user": { "id": "uuid", "username": "john_doe", "email": "john@example.com" },
  		"accessToken": "eyJhbGciOiJIUzI1NiIs...",
  		"refreshToken": "eyJhbGciOiJIUzI1NiIs..."
  	}
  }
  ```

#### 登录

- **POST** `/api/auth/login`
- **请求体**: `{ "email": "john@example.com", "password": "securePassword123" }`
- **成功响应** (200): 返回 `accessToken` 和 `refreshToken`，结构与注册成功相同。

#### 刷新 token

- **POST** `/api/auth/refresh`
- **请求体**: `{ "refreshToken": "eyJhbGciOiJIUzI1NiIs..." }`
- **成功响应** (200):
  ```json
  {
  	"code": 200,
  	"message": "token 已刷新",
  	"data": {
  		"accessToken": "newAccessToken...",
  		"refreshToken": "newRefreshToken..."
  	}
  }
  ```
- **说明**: accessToken 短期有效（如 15 分钟），refreshToken 长期有效（如 7 天），前端在 accessToken 过期时自动调用此接口。

### 4.3 旅程接口

#### 创建旅程

- **POST** `/api/trips`
- **请求体**:
  ```json
  {
  	"title": "京都红叶之旅",
  	"destination": "日本京都",
  	"startDate": "2026-11-01",
  	"endDate": "2026-11-07",
  	"coverImage": "https://example.com/cover.jpg"
  }
  ```
- **响应** (201): 返回完整的 trip 对象。

#### 获取用户所有旅程

- **GET** `/api/trips?page=1&pageSize=10&sort=desc`
- **响应**: 包含分页结构 `{ list, total, page, pageSize }`。

#### 获取单个旅程详情

- **GET** `/api/trips/:id`
- **响应**: 包含 trip 字段及 `expenseCount`、`noteCount`。

#### 更新旅程

- **PATCH** `/api/trips/:id`
- **请求体**: 可部分更新 `title`, `destination`, `startDate`, `endDate`, `coverImage`, `isEnded`。

#### 删除旅程

- **DELETE** `/api/trips/:id`
- **说明**: service 层手动级联删除关联的 expenses 和 notes，同时清理相关图片文件。

### 4.4 账单接口

#### 添加账单（支持小票上传）

- **POST** `/api/trips/:tripId/expenses`
- **Content-Type**: `multipart/form-data`
- **字段**: `category`, `amount`, `expenseDate` (必填), `note`, `receiptImage` (可选)
- **响应** (201): 返回 expense 对象，`receiptImage` 为上传后的文件 URL。

#### 获取旅程的所有账单

- **GET** `/api/trips/:tripId/expenses?category=餐饮&startDate=2026-11-01&endDate=2026-11-07&page=1&pageSize=20`
- **响应**: 支持筛选和分页。

#### 删除账单

- **DELETE** `/api/expenses/:id`

#### 获取消费统计

- **GET** `/api/trips/:tripId/expenses/stats`
- **响应**:
  ```json
  {
  	"code": 200,
  	"message": "success",
  	"data": {
  		"total": 5280.0,
  		"categories": [
  			{ "category": "餐饮", "total": 2150.5, "count": 12 },
  			{ "category": "交通", "total": 800.0, "count": 4 }
  		]
  	}
  }
  ```

### 4.5 游记接口

#### 写游记（支持多图上传 + 自动压缩）

- **POST** `/api/trips/:tripId/notes`
- **Content-Type**: `multipart/form-data`
- **字段**: `content`, `noteDate` (必填), `images` (多文件，可选)
- **处理逻辑**:
  1. 接收图片经 Canvas 压缩后保存至 `uploads/images/`
  2. 原始图片路径数组存入 `images`，`vectorImages` 为预留字段始终为 null
- **响应** (201):
  ```json
  {
  	"code": 201,
  	"message": "游记创建成功",
  	"data": {
  		"id": "uuid",
  		"content": "今天在岚山...",
  		"images": ["/uploads/images/abc.jpg", "/uploads/images/def.jpg"],
  		"noteDate": "2026-11-03",
  		"createdAt": "2026-11-03T12:00:00Z"
  	}
  }
  ```

#### 获取某旅程的游记列表

- **GET** `/api/trips/:tripId/notes?sort=asc`
- **响应**: 数组，每个 note 包含 `images`。

#### 删除游记

- **DELETE** `/api/notes/:id`
- **处理**: 删除数据库记录，同时清理 `uploads/images/` 中的对应文件。

### 4.6 文件上传（独立接口，可选）

- **POST** `/api/upload`
- **请求**: `multipart/form-data` 字段 `file`
- **响应**: `{ "code": 200, "data": { "url": "/uploads/abc123.jpg" } }`

### 4.7 错误码与统一错误处理

- 自定义 `AppError` 类，包含 `statusCode` 和 `message`
- 全局错误中间件捕获所有异常，返回格式 `{ code: statusCode, message: "错误描述" }`
- 生产环境不暴露堆栈信息

| HTTP 状态码 | 含义                | 示例 message     |
| ----------- | ------------------- | ---------------- |
| 400         | 请求参数错误        | "邮箱格式不正确" |
| 401         | 未认证或 token 过期 | "请先登录"       |
| 403         | 无权限操作他人资源  | "无权修改该旅程" |
| 404         | 资源未找到          | "旅程不存在"     |
| 409         | 冲突（如重复注册）  | "用户名已被占用" |
| 500         | 服务器内部错误      | "服务器开了小差" |

## 6. 后端架构设计 (Node.js + Express ESM)

### 6.1 中间件

- **auth**: 验证 accessToken，解析出 `userId`，挂载到 `req.userId`。
- **upload**: Multer 配置，存储至 `uploads/`，限制大小 5MB，类型白名单 jpeg/png/webp。
- **errorHandler**: 全局错误捕获，返回统一格式。

### 6.2 图片处理

游记中上传的图片经客户端 Canvas 压缩（最长边 750px、JPEG 0.8 质量）后保存至 `uploads/images/`，删除游记时同步删除对应的图片文件。

### 6.3 分层结构

- routes -> controllers -> services -> models
- 所有资源修改操作在 service 层校验归属（通过 `userId` 过滤）
- 级联删除在 service 中手动调用子记录的删除逻辑

### 6.4 安全性

- 密码使用 bcrypt 加盐，盐值为 10
- accessToken 有效期 15 分钟，refreshToken 有效期 7 天
- JWT 密钥存储在环境变量中
- CORS 仅允许前端域名
- 使用 `helmet` 增加安全头（推荐）
- 输入校验使用 `express-validator`
